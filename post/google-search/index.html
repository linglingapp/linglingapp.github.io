Google検索とURLに関して

2020年10月13日放送分の「マツコの知らない世界」で，「ハッシュタグの世界」というテーマを取り上げた。Instagramをしないので色々勉強にはなったが，自分が愛してやまないGoogleがディスられている気がした。そこで，Instagramではできないような，検索エンジンならではの検索について書いておきたい。以下では，Googleの「ウェブ検索の精度を高める」を参考に作成するが，そこにはない検索方法もいくつかある。なお，検索演算子の機能は更新される可能性があり，本来は使用できた検索方法が，ある日突然使えなくなったり，新たな検索演算子が使えるようになる場合もある。

完全一致検索

番組では，Googleを利用して「マツコの知らない世界」を検索すると，ワードがバラバラになってヒットすると言っている。

検索語をバラバラにさせないためには「完全一致」を表す「""」を利用すればよい。Googleではこの機能について「単語や語句を引用符で囲みます」と述べている。 つまり，「マツコの知らない世界」を検索するのであれば，"マツコの知らない世界"のようにすればよい。この「""」のような記号を，検索演算子（search operator）という。なお，検索演算子は半角（""）で入力するのが基本である。しかし，Googleの検索エンジンは全角の「””」にも対応している。他の検索エンジンで全角が対応するとは限らないので，注意が必要である。たとえば，Microsoftの検索エンジンBingでは，現在のところ，半角しか対応しない。

site検索

「私はBTSが好きだ」という文の中の「BTS」は何を指しているのだろうか。あなたがARMYならば，「BTS」は間違いなく「防弾少年団」のことであろう。しかし，言語学やコーパスに興味がある人であれば，「BTS」という語は「防弾少年団」以外に，「基本的な文字化の原則（Basic Transcription System）」という頭字語（acronym）を思い浮かべるのかもしれない。

後者の「BTS」を調べるつもりで，Googleで「BTS」とググってみると，前者の「BTS」しか出てこない。後者の「BTS」についてだけ調べたい場合は，site検索を利用してみよう。

「基本的な文字化の原則」は，一般の人々は，おそらくほとんどが何かわからないと思われる。「これを知っていて使っている人は大学に所属している人がほとんどだ」という前提で，site検索の対象をac.jpにしてみよう。つまり，"BTS" site:ac.jpのようにするのである。「ac」は「様々な国に設立されている大学向けの第2レベルドメイン」1である。「jp」は，国別ドメイン（country-code Top-Level Domain; ccTLD)の一種で，Japanの略だから「ac.jp」のようにすると「日本の大学機関」ということになる。

検索結果を見ると，まだ「防弾少年団」が含まれてはいるものの，随所に「防弾少年団」ではない「BTS」についての結果も現れている。もう少し，site検索の対象を絞って「東京外国語大学」のURL（tufs.ac.jp）にしてみよう。"BTS" site:tufs.ac.jpのようにすると，ドメイン名が「tufs.ac.jp」であるサイトが検索対象となる。このようにすると，検索結果に「防弾少年団」が含まれることは（おそらく）ない。

inurl検索

inurl検索は，URLの中に指定した文字列があるのかを確認する。したがって，情報源が「東京外国語大学」（tufs.ac.jp）でなくても，検索結果に現れる場合がある。たとえば，"BTS" inurl:tufs.ac.jpで検索すると，次のURLが引っかかる。

https://b.hatena.ne.jp/entry/www.tufs.ac.jp/ts/personal/usamiken/corpora.htm

これは，「東京外国語大学」のサーバーから発信されているものではない。単にブックマークに登録されているURLの中に「tufs.ac.jp」が含まれていたので引っかかったのである。当然，上記のブックマークは，"BTS" site:tufs.ac.jpで検索すると出てこない。

URLを読む

Google検索をするときに表示されるURLには，どのような意味があるのだろうか2。

最近，「ほぼ」を重複させた「ほぼほぼ」が気になっている。以前，テレビ番組で誰かが「ほぼほぼ100パーセント」と言っていた。この「ほぼ100パーセント」をGoogleで調べてみる。

“ほぼほぼ100パーセント”

検索結果が現れたときのURLを見ると，以下のようになっている。

「ほぼほぼ100パーセント」をGoogleで検索したときのURL

この長ったらしいURLは，何を意味するのだろうか。

日本語のデコード

q

URLの中に「q=」で始まるところがある。以下は，先ほどの長ったらしいURLから，その部分を抜き出したものである。

これは日本語です

%22%E3%81%BB%E3%81%BC%E3%81%BB%E3%81%BC100%E3%83%91%E3%83%BC%E3%82%BB%E3%83%B3%E3%83%88%22&oq=%22%E3%81%BB%E3%81%BC%E3%81%BB%E3%81%BC100%E3%83%91%E3%83%BC%E3%82%BB%E3%83%B3%E3%83%88%22

「 Online Url Decoder」というURLデコーダー（decoder）を利用して，日本語に変換してみる。

これは日本語です

“ほぼほぼ100パーセント”&oq=“ほぼほぼ100パーセント”

ところで，なぜ「“ほぼほぼ100パーセント”」が2回使われているのだろうか。「op=」は何だろう。

op

「op」は，検索窓で私が入力した文字列を表す。先ほど見たように，「op」に与えられた文字列は「q」に与えられた文字列と完全に同じであった。

( ﾟДﾟ)ﾊｧ?

両者の違いを知るために，Googleの自動補完機能を利用してみる。検索窓に「防弾少年団」と入力すると，下の方に「防弾少年団 ダイナマイト」という検索候補が現れる。これをクリックして，「防弾少年団 ダイナマイト」で検索する。つまり，「防弾少年団」を入力したあと，「ダイナマイト」を自分の手で一つずつ入力するのではなく，Googleが教えてくれた検索候補から「防弾少年団 ダイナマイト」を選択するのである。検索結果の画面でURLのとろこを見ると，以下のようになっている。

「q」と「op」が違う

q=%E9%98%B2%E5%BC%BE%E5%B0%91%E5%B9%B4%E5%9B%A3+%E3%83%80%E3%82%A4%E3%83%8A%E3%83%9E%E3%82%A4%E3%83%88&oq=%E9%98%B2%E5%BC%BE%E5%B0%91%E5%B9%B4%E5%9B%A3

わかりずらいけど，URLデコーダーで変換してみると次のようになる。

「検索候補」と「直接入力」の関係

q=防弾少年団 ダイナマイト&oq=防弾少年団

つまり，検索結果のURLに含まれている「q」は「利用者が検索候補を選択して与えた文字列」である。一方，「op」は「利用者が直接入力した文字列」である。検索候補を選択せず，自分が入力した文字列で検索した場合には，検索候補の選択は行われなかったことになるので，「q」と「op」には同じ文字列が与えられるのである。ちなみに，この「q」や「op」のようなものは，パラメーターと呼ばれている。

filter

Googleで検索をすると，最後のページの下段部分で，以下のようなメッセージを目にする。

最も的確な検索結果を表示するために、上の ○○ 件と似たページは除外されています。検索結果をすべて表示するには、ここから再検索してください。

Googleは賢い。しかし，人によっては，この機能が「余計なお世話」になる場合もあるだろう。検索結果を除外せず，すべての結果を表示させるためにはどのようにすればいいのだろうか。

まずは，何がこのように検索結果を制限しているのかを調べてみる。「Google Search Appliance」というものがあるが，それが何なのかは別の機会に説明することにして，Google Search Appliance Helpを見ると，「filter=0」という「query URL parameter」を与えることで，フィルターを通さず，すべての検索結果を表示させることができるらしい。では，これをURLの末尾に付け加えてみよう。

検索結果を除外していないことがわかる。

sxsrf

「sxsrf」が意味するのは，現在の画面が表示される直前の画面に接続したときの日付と時間である。なので，この「sxsrf=ALeKk03TXtnFcu922Dw3-rjPBeZwRy1dAw%3A1606175215577」のところは，なくても問題ない。上記のURLからこの部分を削除した以下のURL接続しても，同じ結果が得られる。

sxsrfを削除したURL

この他にも数種類のパラメーターがある。その一部は，どのような機能があるのか，明らかになっている。しかし，Googleですべてのパラメーターについて説明をしているわけではないので，わからないところもいまだに多い。
