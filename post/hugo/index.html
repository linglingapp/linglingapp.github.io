Hugo와 Google Analytics

2020年6月1日
한국어
717 文字 読了時間: 2 分

Hugo에 Google Analytics을 적용했는데 제대로 작동하지 않았다. 일반적으로는 ‘config.toml’에 있는 googleAnalytics = "" # UA-XXXXXXXX-X라고 적힌 부분에 자신의 tracking ID를 넣어주면 되는 것 같은데 Google Analytics의 ‘실시간’ 탭에서 아무리 확인을 해봐도 사용자 수가 올라가지 않았다. 언제나처럼 삽질을 해 본 결과, Qiita(일본어)에서 해결책을 찾을 수 있었다. 적혀있는 내용을 대충 번역하자면 다음과 같다.

Qiita(@momotaro98)

2020년 5월 3일 현재, Hugo 최신 버전인 v0.69.2에서 Google Analytics과의 연계를 위한 기본 템플릿({{ template “_internal/google_analytics_async.html” . }})은, 최근 사용되고 있는 ‘gtag.js’가 아닌, 예전에 사용되던 ‘analytics.js’을 사용하고 있었습니다.

위 페이지에서는 ‘footer.html’에서 {{ template "_internal/google_analytics_async.html" . }}을 {{- partial "analytics" . -}}로 바꿔주었다. 현재 내가 사용하고 있는 테마는 even이다. 이 테마에서는 해당부분이 아래 경로의 ‘scripts.html’에 있다.

C:\hugo\themes\even\layouts\partials\scripts.html

파일을 열어보면 아래와 같이 되어 있는데, {{- partial "analytics" . -}}로 바꿔주자.

<!-- Analytics -->
{{- if (in (slice (getenv "HUGO_ENV") hugo.Environment) "production") | and .Site.GoogleAnalytics -}}
  {{ template "_internal/google_analytics_async.html" . }}
{{- end -}}

 {{ if not .Site.IsServer }}
 {{ with .Site.GoogleAnalytics }}
 <!-- Global site tag (gtag.js) - Google Analytics -->
 <script async src="https://www.googletagmanager.com/gtag/js?id={{ . }}"></script>
 <script>
   window.dataLayer = window.dataLayer || [];
   function gtag(){dataLayer.push(arguments);}
   gtag('js', new Date());

   gtag('config', '{{ . }}');
 </script>
 {{ end }}
 {{ end }}
